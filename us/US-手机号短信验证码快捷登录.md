# US 设计文档：手机号 + 短信验证码快捷登录

## 1. 背景与目标

为降低用户登录门槛、提升转化率与回访率，系统新增“手机号 + 短信验证码”快捷登录能力。该能力同时覆盖：

- 新用户：首次使用手机号登录时自动完成注册；
- 已注册用户：使用验证码直接登录；
- 记住我：用户可选择在当前设备保持登录状态，减少重复验证。

## 2. 用户故事（User Story）

作为一名用户，
我希望使用手机号和短信验证码快速登录，
以便无需记忆密码也能安全地访问系统；
并且我可以选择“记住我”，让下次访问更便捷。

## 3. 范围（Scope）

### 3.1 In Scope

1. 手机号输入与格式校验（中国大陆 11 位手机号）。
2. 短信验证码发送、重发倒计时、频控与有效期管理。
3. 验证码登录流程：
   - 手机号已存在 → 登录成功；
   - 手机号不存在 → 自动注册并登录成功。
4. “记住我”选项：延长登录态有效时长。
5. 基础风控与安全策略（限流、防刷、错误次数控制）。
6. 关键埋点与日志（发送成功率、登录转化、失败原因）。

### 3.2 Out of Scope

1. 国际手机号支持。
2. 语音验证码。
3. 第三方社交账号绑定登录。
4. 图形验证码（仅在风控升级阶段再引入）。

## 4. 业务规则

1. 验证码为 6 位数字。
2. 验证码有效期：5 分钟。
3. 同一手机号发送间隔：60 秒。
4. 同一手机号每日发送上限：20 次。
5. 同一 IP 每小时发送上限：100 次（可配置）。
6. 验证码连续输错 5 次后，手机号登录入口临时锁定 15 分钟。
7. 自动注册时需创建最小用户档案（手机号、注册时间、来源渠道）。
8. “记住我”勾选时：登录态有效期 30 天；未勾选时：登录态有效期 24 小时。

## 5. 交互流程

### 5.1 登录页

字段与控件：
- 手机号输入框
- 验证码输入框
- 「发送验证码」按钮
- 「记住我」复选框（默认不勾选）
- 「登录 / 注册」主按钮

### 5.2 发送验证码流程

1. 用户输入手机号。
2. 前端校验格式合法。
3. 调用发送验证码接口。
4. 服务端通过频控、风控校验后下发短信。
5. 前端展示“已发送”，并进入 60 秒倒计时，期间不可重复发送。

### 5.3 登录 / 注册流程

1. 用户输入验证码并提交。
2. 服务端校验验证码正确性、有效期与错误次数。
3. 根据手机号查询用户：
   - 已存在：登录成功。
   - 不存在：创建用户并登录成功。
4. 按“记住我”策略签发会话令牌。
5. 返回登录态并跳转目标页面。

## 6. 接口设计（建议）

### 6.1 发送验证码

POST /api/v1/auth/sms/send

Request:
```json
{
  "phone": "13800138000",
  "scene": "login"
}
```

Response:
```json
{
  "success": true,
  "request_id": "req_xxx",
  "retry_after": 60
}
```

错误码示例：
- AUTH_PHONE_INVALID
- AUTH_SMS_RATE_LIMIT
- AUTH_SMS_DAILY_LIMIT
- AUTH_RISK_BLOCKED

### 6.2 验证码登录（含自动注册）

POST /api/v1/auth/sms/login

Request:
```json
{
  "phone": "13800138000",
  "code": "123456",
  "remember_me": true
}
```

Response:
```json
{
  "success": true,
  "is_new_user": false,
  "user": {
    "id": "u_10001",
    "phone": "13800138000"
  },
  "token": {
    "access_token": "xxx",
    "expires_in": 2592000
  }
}
```

错误码示例：
- AUTH_CODE_INVALID
- AUTH_CODE_EXPIRED
- AUTH_CODE_RETRY_EXCEEDED
- AUTH_LOGIN_TEMP_LOCKED

## 7. 数据模型变更（建议）

### 7.1 user 表

新增 / 确认字段：
- phone（唯一索引）
- register_source（sms_login）
- created_at / updated_at

### 7.2 sms_code_log 表

字段建议：
- id
- phone
- scene
- code_hash（不明文存储）
- expired_at
- send_ip
- verify_fail_count
- status（sent / used / expired / locked）
- created_at

### 7.3 login_session 表（或 token 黑白名单机制）

字段建议：
- user_id
- device_id（可选）
- remember_me
- issued_at
- expires_at
- revoked_at

## 8. 安全与风控

1. 验证码仅存储 hash，不落明文。
2. 登录与发送接口需接入限流（手机号、IP、设备维度）。
3. 高风险行为（短时高频请求、异常 IP）触发拦截或二次验证。
4. 短信模板内容避免泄露业务敏感信息。
5. 关键操作日志可审计，便于追溯与风控调参。

## 9. 埋点与监控

### 9.1 埋点事件

- auth_sms_send_click
- auth_sms_send_success
- auth_sms_send_fail
- auth_sms_login_submit
- auth_sms_login_success
- auth_sms_login_fail
- auth_sms_auto_register_success
- auth_remember_me_checked

### 9.2 核心指标

1. 验证码发送成功率。
2. 发送到提交验证码转化率。
3. 登录成功率。
4. 自动注册占比。
5. 次日留存（记住我 vs 未勾选对比）。

## 10. 验收标准（Acceptance Criteria）

1. 用户输入合法手机号后可成功发送验证码，60 秒内不可重复发送。
2. 验证码在 5 分钟内输入正确可登录成功。
3. 未注册手机号首次登录后自动创建账号并返回登录态。
4. 已注册手机号登录不创建新账号，直接返回登录态。
5. 勾选“记住我”后会话有效期为 30 天；未勾选为 24 小时。
6. 验证码错误超过阈值后触发临时锁定并返回明确提示。
7. 关键埋点可在分析系统中被正确采集。

## 11. 测试用例（核心）

1. 正常路径：发送验证码 → 输入正确验证码 → 登录成功。
2. 新用户路径：不存在手机号 → 输入正确验证码 → 自动注册成功。
3. 验证码过期：超过 5 分钟后提交 → 返回过期错误。
4. 连续输错：达到 5 次后 → 触发锁定。
5. 频控验证：60 秒内重复发送 → 被拒绝。
6. remember_me=true/false 下 token 过期时间正确。
7. 同手机号并发请求下不会创建重复用户（幂等性）。

## 12. 发布与回滚

### 发布步骤
1. 后端接口上线（灰度）
2. 前端入口灰度放量
3. 监控发送成功率、登录成功率、错误码分布

### 回滚策略
1. 关闭短信登录入口开关（feature flag）
2. 保留密码登录兜底
3. 回退到上一个稳定版本

## 13. 风险与待确认项

1. 短信通道成本与到达率波动。
2. 海外用户是否在后续阶段支持。
3. 是否在高风险场景叠加图形验证码。
4. remember_me 在多设备并发下的会话策略（是否互踢）。
