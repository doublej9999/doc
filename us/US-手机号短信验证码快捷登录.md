## 1. 需求背景
当前登录流程对“记忆密码”依赖较高，影响首登与回访转化。为降低登录门槛，需要提供“手机号 + 短信验证码”快捷登录方式，并在同一路径中兼容：
1) 新用户自动注册并登录；
2) 老用户直接登录；
3) 提供“记住我”能力减少重复登录。

---

## 2. 用户故事（User Story）
作为用户，
我希望通过手机号和短信验证码快速登录系统，
这样我不需要记住密码也能完成访问；
并且我可以选择“记住我”，在常用设备上保持更长登录状态。

---

## 3. 目标与非目标

### 3.1 目标（Goals）
- G1：支持手机号验证码登录主流程。
- G2：未注册手机号可自动完成注册并登录。
- G3：已注册手机号可直接登录。
- G4：支持“记住我”并影响会话有效期。
- G5：具备基础风控、限流与审计日志。

### 3.2 非目标（Non-goals）
- N1：本期不支持海外手机号。
- N2：本期不支持语音验证码。
- N3：本期不引入第三方社交登录。
- N4：本期不做复杂设备指纹风控（预留扩展点）。

---

## 4. 角色与场景

### 4.1 角色
- 访客用户（未注册）
- 注册用户（已存在账号）
- 风控系统
- 短信服务商

### 4.2 场景
- S1：新用户首次输入手机号并验证成功，自动注册+登录。
- S2：老用户输入手机号并验证成功，直接登录。
- S3：用户勾选“记住我”，获得更长会话有效期。
- S4：频繁发送/错误输入触发限流或临时锁定。

---

## 5. 业务规则
1. 手机号规则：中国大陆 11 位手机号格式校验。
2. 验证码规则：6 位数字随机码。
3. 验证码有效期：5 分钟。
4. 发送间隔：同手机号 60 秒内不可重复发送。
5. 发送上限：同手机号每日最多 20 次（可配置）。
6. IP 发送限制：同 IP 每小时最多 100 次（可配置）。
7. 错误限制：同手机号验证码连续错误 5 次，锁定 15 分钟。
8. 自动注册：验证码通过后若手机号未注册，自动创建账号。
9. 记住我：
   - 勾选：会话有效期 30 天
   - 不勾选：会话有效期 24 小时
10. 幂等要求：并发自动注册同一手机号时，保证仅创建一个用户。

## 6. 交互与流程设计

### 6.1 页面元素
- 手机号输入框
- 验证码输入框
- 「发送验证码」按钮（带倒计时）
- 「记住我」复选框（默认不勾选）
- 「登录 / 注册」提交按钮

### 6.2 主流程（Happy Path）
1. 用户输入手机号。
2. 点击发送验证码，前端先做格式校验。
3. 后端完成限流/风控校验后发送短信。
4. 用户输入验证码并点击登录。
5. 后端校验验证码正确性与时效。
6. 若手机号存在：直接登录。
7. 若手机号不存在：自动注册后登录。
8. 根据 remember_me 签发会话并返回。

### 6.3 异常流程
- A1：手机号格式错误 → 前端拦截。
- A2：发送过于频繁 → 返回限流提示及剩余等待时间。
- A3：验证码错误/过期 → 提示重试或重新发送。
- A4：错误次数超限 → 临时锁定并提示锁定时长。
- A5：短信通道异常 → 友好提示“发送失败，请稍后重试”。

## 7. 接口设计（US级）

### 7.1 发送验证码
POST `/api/v1/auth/sms/send`

Request:
```json
{
  "phone": "13800138000",
  "scene": "login"
}
```

Response:
```json
{
  "success": true,
  "request_id": "req_abc123",
  "retry_after": 60
}
```

错误码建议：
- `AUTH_PHONE_INVALID`
- `AUTH_SMS_RATE_LIMIT`
- `AUTH_SMS_DAILY_LIMIT`
- `AUTH_RISK_BLOCKED`
- `AUTH_SMS_PROVIDER_ERROR`

### 7.2 验证码登录（含自动注册）
POST `/api/v1/auth/sms/login`

Request:
```json
{
  "phone": "13800138000",
  "code": "123456",
  "remember_me": true
}
```

Response:
```json
{
  "success": true,
  "is_new_user": true,
  "user": {
    "id": "u_1024",
    "phone": "13800138000"
  },
  "token": {
    "access_token": "token_xxx",
    "expires_in": 2592000
  }
}
```

错误码建议：
- `AUTH_CODE_INVALID`
- `AUTH_CODE_EXPIRED`
- `AUTH_CODE_ATTEMPTS_EXCEEDED`
- `AUTH_LOGIN_TEMP_LOCKED`

## 9. 安全与风控要求
1. 验证码仅存 hash，不可逆存储。
2. 接口全链路 HTTPS。
3. 发送/登录接口均需限流（手机号、IP维度）。
4. 关键行为写入审计日志（发送、验证、锁定、解锁）。
5. 防重放：同验证码仅允许成功使用一次。
6. 预留风控策略开关（黑名单、异常地域拦截）。

## 11. 验收标准（Acceptance Criteria）
1. 合法手机号可发送验证码，60 秒内不可重发。
2. 验证码 5 分钟内正确提交可登录成功。
3. 未注册手机号通过验证码后自动注册并登录。
4. 已注册手机号通过验证码后直接登录，不创建新用户。
5. remember_me=true 时会话有效期为 30 天；false 为 24 小时。
6. 连续错误 5 次触发 15 分钟锁定并有可读提示。
7. 埋点事件在分析平台可查询，字段完整。

## 12. 测试清单（核心）
- T1：发送验证码成功与倒计时校验
- T2：验证码正确登录（老用户）
- T3：验证码正确登录（新用户自动注册）
- T4：验证码过期
- T5：验证码连续错误触发锁定
- T6：发送频控与日限额
- T7：remember_me 会话时长差异
- T8：并发自动注册幂等（无重复账号）

## 13. 发布与回滚

### 13.1 发布策略
1. 后端接口先灰度。
2. 前端入口按用户比例放量。
3. 监控成功率、失败码、投诉量。

### 13.2 回滚策略
1. 关闭短信登录开关（Feature Flag）。
2. 保留密码登录兜底。
3. 回滚到上一个稳定版本。
