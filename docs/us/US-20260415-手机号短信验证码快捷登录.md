# Story ID
US-20260415-001

# 标题
手机号 + 短信验证码快捷登录（含新用户自动注册与“记住我”）

# 描述
为提升登录转化率与使用便捷性，系统新增“手机号 + 短信验证码”快捷登录能力。用户输入手机号并通过短信验证码校验后即可完成登录；若手机号未注册，系统自动完成账号注册并登录；若已注册，则直接登录。同时提供“记住我”选项，用于控制登录态保持时长，减少重复登录成本。

# 验收标准
- 支持中国大陆 11 位手机号输入校验，非法手机号不可发送验证码。
- 验证码发送具备频控：同手机号发送间隔 ≥ 60s，24 小时上限可配置（默认 20 次）。
- 验证码有效期可配置（默认 5 分钟），超时后不可使用。
- 验证码校验通过后：
  - 未注册手机号：自动创建用户并登录。
  - 已注册手机号：直接登录。
- 登录页提供“记住我”勾选项：
  - 勾选：登录态有效期 30 天（可配置）。
  - 不勾选：登录态有效期 24 小时（可配置）。
- 登录成功后返回统一用户信息与 token；失败返回标准错误码与提示文案。
- 全链路日志可追踪（发送、校验、登录、注册），敏感信息脱敏。

Given-When-Then:
1) Given 用户输入合法且未注册手机号，When 验证码校验成功，Then 系统自动注册账号并返回登录成功。
2) Given 用户输入合法且已注册手机号，When 验证码校验成功，Then 系统直接登录并返回登录成功。
3) Given 用户勾选“记住我”，When 登录成功，Then 会话有效期为 30 天。
4) Given 用户未勾选“记住我”，When 登录成功，Then 会话有效期为 24 小时。
5) Given 验证码错误或过期，When 提交登录，Then 返回失败且不创建会话。

# 概要设计
采用“验证码校验成功后统一颁发会话”的架构：
- 认证服务（Auth Service）新增短信验证码发送与校验接口。
- 用户服务（User Service）提供手机号查询与自动注册能力。
- 会话服务（Session Service）根据 remember_me 参数下发不同 TTL 的 access/refresh token。
- 短信网关（SMS Provider）负责发送短信，认证服务维护发送状态与频控策略。

流程概览：
1) 用户请求发送验证码 -> Auth 校验手机号与频控 -> 调用 SMS 网关发送 -> 存储验证码摘要与有效期。
2) 用户提交手机号 + 验证码 + remember_me -> Auth 校验验证码 -> 查询用户：
   - 不存在：调用 User Service 创建用户；
   - 存在：直接继续；
3) Session Service 颁发 token（TTL 由 remember_me 决定）-> 返回登录成功。

# 详细设计
## 1. 验证码发送
- 入参：手机号、业务场景（login）。
- 校验：手机号格式、图形验证码（可选）、设备/IP 风控。
- 频控策略：
  - 手机号级：60s 内不可重复发送；
  - 手机号日限：默认 20 次；
  - IP 日限：默认 200 次。
- 存储：验证码不明文存储，使用 HMAC(phone+code+salt) 保存摘要；记录过期时间、尝试次数、发送流水号。

## 2. 验证码登录
- 入参：手机号、验证码、remember_me。
- 校验：验证码是否存在、是否过期、是否超过尝试次数（默认 5 次）。
- 用户判定：按手机号查询用户，不存在则自动注册。
- 注册策略：
  - 生成最小用户资料（手机号、默认昵称、注册来源=SMS_LOGIN）。
  - 幂等控制：手机号唯一索引 + 事务重试，避免并发重复注册。
- 会话策略：
  - remember_me=true：refresh_token_ttl=30d；
  - remember_me=false：refresh_token_ttl=24h；
  - access_token_ttl 统一 2h（可配置）。

## 3. 安全策略
- 验证码单次有效，校验成功后立即失效。
- 多次错误后短时锁定手机号（默认 15 分钟）。
- 敏感字段脱敏日志（手机号中间四位脱敏）。
- 防重放：验证码与 request_id/nonce 绑定（可选增强）。

## 4. 兼容与迁移
- 不影响现有账号密码登录。
- 支持灰度开关：按环境/租户/用户分组开启。

{

## 性能
- 登录接口 P95 < 300ms（不含短信发送）。
- 发送接口 P95 < 500ms（受供应商网络波动影响）。

## 安全
- 验证码摘要存储 + 单次失效。
- 限流、防刷、防爆破。
- Token 签发与刷新遵循现有安全策略（签名、过期、吊销）。

## 可扩展性
- 支持多国家区号。
- 可扩展为语音验证码、WhatsApp OTP。
- 支持多短信供应商路由与故障切换。

## 监控
- 指标：发送成功率、校验成功率、登录成功率、自动注册占比、风控拦截率。
- 告警：发送失败率超过阈值、登录成功率突降、单 IP 异常高频。

# 风险与依赖
风险：
1) 短信通道不稳定导致发送延迟/失败。
2) 攻击者恶意刷短信引发成本与风控压力。
3) 多端并发登录导致会话管理复杂度上升。

依赖：
1) 第三方短信服务商能力与 SLA。
2) 用户服务手机号唯一约束与幂等保障。
3) 会话服务支持动态 TTL。
